# Weekly Sprint Planning

## When to Use
Every Monday after the client health check. 20 minutes.

## Process

### 1. REVIEW LAST WEEK
- What shipped? (features, client deliverables, fixes)
- What didn't ship? Why? (blocked, deprioritized, underestimated)
- Any surprises? (client escalations, bugs, scope changes)

### 2. THIS WEEK'S PRIORITIES
Categorize everything into:

**P0 — Client Work (revenue)**
- Active client deliverables with deadlines
- Client-reported issues
- Monthly reports due

**P1 — Sales (pipeline)**
- Outreach emails to send
- Proposals to write
- Discovery calls to prep for
- Follow-ups due

**P2 — Product (framework)**
- Bug fixes from audit report
- Features that would help client work
- Week-2 items from launch audit

**P3 — Growth (marketing)**
- Blog posts
- LinkedIn content
- Case study documentation
- Website improvements

### 3. COMMIT TO 3 THINGS
Pick exactly 3 outcomes for the week:
1. One client deliverable
2. One sales action
3. One product/growth task

Three is the max. Anything else is bonus. Focus beats breadth.

### 4. SET UP RALPH (if applicable)
If any P2 product work involves multiple PRD items:
- Update PRD.md with this week's items
- Run Ralph loop for autonomous execution
- Review results at end of day

## Output
```
WEEK OF [DATE]

COMMITTED:
1. [Client deliverable] — due [day]
2. [Sales action] — target [number]
3. [Product/growth] — ship by [day]

BACKLOG (if time permits):
- [item]
- [item]
```
